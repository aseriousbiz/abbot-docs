    IPattern

    The pattern that caused a skill to be invoked. Most skills are called by name. For example, by mentioning Abbot followed by the skill name, or by using the shortcut character followed by the skill name. For example, 

    .help
    calls the 
    help
    skill.

    A pattern is another way a skill can be called. When a message that is NOT an Abbot command matches a skill's pattern, the skill is called with the message as the arguments. This interface describes such a pattern.
